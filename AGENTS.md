# Agent & contributor conventions — Gancho

- **`project.yml` is the source of truth.** Never hand-edit `Gancho.xcodeproj`
  (it is generated and git-ignored). After changing `project.yml`, run
  `make project`.
- **Swift 6 strict concurrency, no exceptions.** App targets default to
  `@MainActor` (approachable concurrency); the engine-room modules in
  `Packages/GanchoKit` are `nonisolated + Sendable` — annotate isolation per
  type only when one genuinely needs it. Adding a new top-level package under
  `Packages/` requires extending the Makefile's `PACKAGE` wiring so the gates
  cover it.
- **No deprecated APIs.** Minimum OS is macOS 26 / iOS 26; new SDK-27 APIs go
  behind `#available`. Do not install beta SDKs on work machines.
- **Run `make format` before committing; `make lint` and `make test` must pass.**
- **Tests:** Swift Testing (`@Suite`/`@Test`) for unit tests. XCTest only for
  future UI tests. Accessibility identifiers use kebab-case with semantic
  suffixes (`history-panel`, `clip-row`, `settings-pane`).
- **Commits:** Conventional Commits, English, imperative. **Never add AI
  co-authorship or generated-by trailers** (no `Co-Authored-By: Claude`, no
  watermarks) — repository policy.
- **Prose:** all project documentation is English unless the maintainer
  explicitly asks for another language. Chat may be Spanish, but docs, local
  planning files, code comments, commits, and PRs stay English by default.
  User-facing strings will live in a String Catalog with English + Spanish from
  the first UI string (enforced by a LocalizationTests gate, pattern inherited
  from vitrine).
- **Privacy is a hard rule, not a feature flag.** Never store content carrying
  `org.nspasteboard.ConcealedType` / `TransientType` / `AutoGeneratedType`.
  No clipboard content in logs, telemetry, or error reports — ever.
- **Product decisions live in the maintainer's local planning docs**
  (`.planning/`, git-ignored: backlog with acceptance criteria, pricing,
  naming history). Do not re-litigate them in code comments. Committed text —
  code comments, commit messages, PRs, docs — never references internal ticket
  IDs or tracker names; describe the functionality instead.
